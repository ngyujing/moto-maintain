Hosting & Database Setup

Two parts: host the app on GitHub Pages (free, permanent URL), then set up the Appwrite database automatically with the Node script.
Total time: ~20 minutes.

🐙
Part A — GitHub Pages
Host index.html at a free public URL like yourusername.github.io/moto-maintain
A1
Install Git on your computer
One-time setup. Skip if you already have Git.

Open a terminal (Mac: Terminal app; Windows: search "Command Prompt" or "PowerShell"; Linux: any terminal) and run:

git --version

If you see a version number like git version 2.43.0, skip to A2. If not, install it:

OSHow to install
macOSRun git --version — macOS will automatically prompt to install Developer Tools. Click Install.
WindowsGo to git-scm.com/download/win → download the installer → run it → click Next through all defaults.
Ubuntu / Debiansudo apt update && sudo apt install git -y

After installing, configure your identity (this labels your commits — required by Git):

git config --global user.name  "Your Full Name"
git config --global user.email "you@example.com"
💡
Use the same email address as your GitHub account. This links your commits to your profile.
A2
Create the GitHub repository
A repository is the online folder that stores your code.
  1. 1Go to github.com and sign in (or create a free account if you don't have one).
  2. 2Click the ＋ icon in the top-right corner → New repository.
  3. 3Set Repository name to moto-maintain. This becomes part of your URL.
  4. 4Set visibility to Public. (GitHub Pages requires Public on the free plan.)
  5. 5Leave all checkboxes unchecked. Do NOT add a README, .gitignore, or licence file — adding them now causes a merge conflict in the next step.
  6. 6Click Create repository. You'll land on an empty repo page — keep this tab open.

Your repository URL will look like: https://github.com/yourusername/moto-maintain

A3
Put index.html in a folder and push to GitHub
GitHub Pages serves files directly from your repository.

1. Create a project folder and place index.html inside it:

# Create the folder anywhere you like
mkdir moto-maintain
cd moto-maintain

# Copy your index.html into this folder
# (or move it manually using Finder / File Explorer)

2. Initialise Git and make your first commit:

git init
git add index.html
git commit -m "Add MotoMaintain app"

3. Connect to GitHub and push. Replace yourusername with your GitHub username:

git remote add origin https://github.com/yourusername/moto-maintain.git
git branch -M main
git push -u origin main

Git will ask for your GitHub credentials. For the password, you cannot use your account password — you need a Personal Access Token:

  1. 1On GitHub, click your profile photo → Settings → scroll to bottom → Developer settings.
  2. 2Click Personal access tokens → Tokens (classic) → Generate new token (classic).
  3. 3Note: moto-maintain · Expiration: 90 days · Scope: tick repo only → click Generate token.
  4. 4Copy the token immediately — it's shown once. Paste it as the password when Git prompts.
✅
You should see output like Branch 'main' set up to track remote branch 'main' from 'origin'. Your file is now on GitHub.
A4
Enable GitHub Pages
Turns your repository into a live public website in about 60 seconds.
  1. 1Go to your repository on GitHub: github.com/yourusername/moto-maintain.
  2. 2Click the Settings tab (the gear icon, far right of the tab row).
  3. 3In the left sidebar, scroll down and click Pages.
  4. 4Under Source, click the dropdown that says "None" and select Deploy from a branch.
  5. 5Under Branch, select main and leave the folder as / (root). Click Save.
  6. 6Wait about 60 seconds, then refresh the Pages settings page. You'll see a green banner with your URL.

Your live URL will be:

https://yourusername.github.io/moto-maintain/
⏱
The first deploy takes 1–3 minutes. Subsequent updates (after git push) are usually live within 30 seconds. If you get a 404, wait a minute and refresh.

To verify it worked, go to Actions tab in your repo. You'll see a workflow called pages build and deployment with a green tick once it's live.

🔄
Every time you update the app and push with git add . → git commit → git push, GitHub Pages automatically rebuilds and your live URL updates within a minute.
A5
Optional — Use a custom domain
Point your own domain (e.g. motomaintain.com) to GitHub Pages.

If you own a domain, you can point it to your GitHub Pages site:

  1. 1In your domain registrar's DNS settings, add a CNAME record: www → yourusername.github.io.
  2. 2For the root domain (no www), add 4 A records pointing to GitHub's IPs: 185.199.108.153, 185.199.109.153, 185.199.110.153, 185.199.111.153.
  3. 3Back in GitHub → Settings → Pages → Custom domain → type your domain → Save.
  4. 4Tick Enforce HTTPS once it appears (may take 10–30 minutes for DNS to propagate).
💡
DNS propagation takes 10 minutes to 48 hours depending on your registrar. Don't be alarmed if your domain doesn't work immediately.
🗄️
Part B — Appwrite Database (Node Script)
Automatically creates all 5 collections, indexes, and storage bucket with one command
B1
Create an Appwrite project
Appwrite Cloud is free. No credit card required.
  1. 1Go to cloud.appwrite.io and sign up for a free account (or log in).
  2. 2Click Create Project → enter name MotoMaintain → click Create.
  3. 3On the next screen it asks to add a platform. Click Web app.
  4. 4For Name type MotoMaintain Web. For Hostname type localhost (you can add your GitHub Pages domain later).
  5. 5Click Next through the remaining screens — you don't need to install an SDK now.
  6. 6On the project Overview page, find the Project ID — it's a short string like 67a3f9b2c1d4e. Copy it.
📋
The Project ID is on the Settings page too: Project → Settings → General → copy the Project ID field.
B2
Create a Server API Key
The Node script needs a key with full database access to create collections.
  1. 1In your Appwrite project, click Settings in the left sidebar.
  2. 2Click API Keys → Create API key.
  3. 3Name it MotoMaintain Setup Key.
  4. 4Under Add scope, expand each section and tick everything under: Databases, Users, Storage.
  5. 5Set Expiration to 1 day (you only need it for setup).
  6. 6Click Create. Copy the key immediately — it's only shown once.
🔐
Never paste this API key into your index.html or commit it to GitHub. It has admin access to your entire Appwrite project.
B3
Set up the project folder and .env file
The Node script reads credentials from a local .env file — never hardcoded.

Make sure Node.js is installed. Check with:

node --version
# Should show v18 or higher
npm --version

If Node is not installed, download it from nodejs.org → click the LTS version → run the installer.

Inside your moto-maintain folder, create a backend subfolder and set it up:

cd moto-maintain
mkdir backend
cd backend
npm init -y
npm install node-appwrite dotenv

Now create the .env file. Open a text editor, create a file named .env inside backend/, and fill it in:

# backend/.env — never commit this file
APPWRITE_ENDPOINT=https://cloud.appwrite.io/v1
APPWRITE_PROJECT_ID=paste_your_project_id_here
APPWRITE_API_KEY=paste_your_api_key_here
APPWRITE_DATABASE_ID=moto_maintain_db
STORAGE_BUCKET_ID=service_attachments

Create a .gitignore in the backend/ folder so the key is never uploaded:

# backend/.gitignore
node_modules/
.env
🚨
Triple-check that .env is in your .gitignore before running git add. Run git status and confirm .env does not appear in the list.
B4
Place the setup script and run it
The script creates all 5 collections, attributes, indexes, and the storage bucket.

Copy the appwrite-setup.js file into your backend/ folder. Your folder structure should now look like:

moto-maintain/
├── index.html                 ← your app
└── backend/
    ├── appwrite-setup.js      ← the setup script
    ├── .env                   ← your credentials (never commit)
    ├── .gitignore
    ├── package.json
    └── node_modules/

Now run the setup script from inside backend/:

cd backend
node appwrite-setup.js

Watch the output. You should see each collection being created:

⏱
The script adds a small delay between attribute creations. It takes about 60–90 seconds total. This is normal — Appwrite needs time to activate each attribute before indexes can be built on top of them.
B5
Seed service templates
Populates the service_templates collection with default intervals per bike type.

Copy seed.js into backend/ and run it:

node seed.js

Expected output:

🌱 MotoMaintain Seed Script
──────────────────────────────
✅ Created: Engine Oil (5,000 km / 180 days)
✅ Created: Oil Filter (5,000 km / 180 days)
✅ Created: Air Filter (15,000 km / 365 days)
... (12 total)
📊 Results: 12 created, 0 skipped
✅ Seed complete!
🔁
The seed script is safe to re-run. It checks if each template already exists and skips duplicates — you'll see ⏭ Skipping for existing ones.
B6
Verify everything in Appwrite Console
Confirm all collections, attributes, and indexes were created correctly.

Go to cloud.appwrite.io → your project → Databases → MotoMaintain DB. You should see 5 collections:

Collection IDDocumentsStatus
motorcycles0✓ Created
maintenance_items0✓ Created
service_records0✓ Created
service_templates12✓ Seeded
odometer_logs0✓ Created

Go to Storage — you should see a bucket called Service Attachments.

🌐
Now update your index.html with your Project ID (in the Appwrite settings section of the app) and push to GitHub. Your live Pages URL will connect to this database.
⚠️
Add your live GitHub Pages URL to Appwrite's allowed platforms: Settings → Platforms → Add Platform → Web → enter yourusername.github.io as the hostname. Otherwise Appwrite will block requests from your live URL.
Common errors
Error messageFix
401 UnauthorizedYour APPWRITE_API_KEY is wrong or expired. Generate a new one in Appwrite Settings → API Keys.
Project not foundYour APPWRITE_PROJECT_ID is wrong. Copy it from Appwrite Settings → General.
Collection already existsSafe — the script skips existing collections. Run again; it won't duplicate anything.
Cannot find module 'node-appwrite'Run npm install inside the backend/ folder first.
ENOENT: .env not foundMake sure .env is in the backend/ folder, not the parent folder.
🎉

You're live!

Your app is hosted at yourusername.github.io/moto-maintain and connected to Appwrite. Push any future changes with git add . → git commit → git push and they go live automatically.
